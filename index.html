‚ú® Project Abstract: Centralized Reading Data

BOOK Hive addresses the fragmentation of online book content by creating a **secure, centralized relational database** using MySQL. Our objective is to move user-generated content (reviews and summaries) from unstructured sources into a highly organized system that enforces **data integrity** between user profiles, book records, and posts. This robust foundation supports efficient queries and powerful community analytics.

‚öôÔ∏è Core Relational Design (User, Book, Post)

The system is built on a highly normalized schema featuring clear **one-to-many (1:N)** relationships. This structure ensures every post is reliably linked to an authenticated author and a valid book entry, preventing 'orphaned' content.

Post Table DDL Example

CREATE TABLE Post (
  post_id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL, /* FK to User */
  book_isbn CHAR(13) NOT NULL, /* FK to Book */
  post_type ENUM('Summary', 'Review') NOT NULL,
  content_text TEXT NOT NULL,
  rating INT,
  FOREIGN KEY (user_id) REFERENCES User(user_id) ON DELETE CASCADE,
  FOREIGN KEY (book_isbn) REFERENCES Book(book_isbn) ON DELETE RESTRICT
);
